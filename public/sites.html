Connection Workflow

Create or select a site, then complete each step in order.

1

Identify the site

Use a short stable Site ID. This is how the collector tracks the connection and saved alarms.

2

Enter the gateway details

Enter the HTTPS base URL reachable through BrinkAgent/Neeve. The app will append the alarm path during testing.

3

Test the connection

This verifies the server can reach the URL and authenticate. A zero alarm count can still be valid if no active alarms exist.

4

Fetch and review alarms

Use this after the connection test succeeds. It saves matching alarms into MongoDB and refreshes the table below.
